# Introduction

目前这些资源是参照6.0的版本开发的。  

引入ELK，目标不是降低日志存储空间，相反一定会增加日志存储空间。  
引入ELK的目标是便于日志分析（连管理功能都没有），继而产生各种附加值、衍生品，比如故障通知、故障分析、性能分析、性能估算和预警。
